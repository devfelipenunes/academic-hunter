AI-Native Research via MCP

Academic Hunter is not just a Python CLI — it is a native Model Context Protocol (MCP) server. Any MCP-compatible client can drive the full research pipeline autonomously.

MCP is a universal connector between AI and tools. Instead of switching between apps, your AI assistant calls Academic Hunter directly — configuring searches, exploring citation graphs, running semantic queries, and saving findings to your second brain. All in real time.

Add to Claude / Cursor

{
  "mcpServers": {
    "academic-hunter": {
      "command": "/path/to/venv/bin/python",
      "args": ["-m", "academic_hunter.interfaces.mcp.server"],
      "env": { "PYTHONPATH": "/path/to/src" }
    }
  }
}

The auto-installer (python install.py) does this for you.

How It Works

From configuration to published results in four steps.

Configure

Define research anchors, technical terms, and scoring weights in config.json. The engine uses these as its compass.

Mine

The hunter sweeps all 7 academic APIs in parallel. Papers are fetched, deduplicated, and enriched with missing metadata.

Score

Every paper is evaluated by keyword and semantic screeners. Only those above your min_relevance_score threshold survive.

Report

Results are exported as rich reports (CSV, BibTeX, Markdown with PRISMA flow) and indexed into ChromaDB for semantic queries.

Quickstart

Clone, install, and start hunting in under two minutes.

$ git clone https://github.com/devfelipenunes/academic-hunter.git
$ cd academic-hunter
$ python install.py
$ source venv/bin/activate
$ academic-hunter
$ # Or, for MCP server mode:
$ academic-mcp
