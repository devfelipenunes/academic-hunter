Features

Built for systematic rigor. Designed for automation.

⚙

Multi-Source Aggregation

Concurrently queries Semantic Scholar, OpenAlex, Crossref, ArXiv, CORE.ac.uk, DBLP, and DOAJ — no paper left behind.

★

Elite Scoring Engine

NLP and keyword evaluators mathematically score each paper against your research anchors. Configurable weights down to the term level.

↯

Smart Deduplication

Strict-DOI and fuzzy title matching eliminate duplicates across sources, keeping your dataset clean and defensible.

◎

Native MCP Server

15 tools accessible to any MCP client. Claude, Cursor, or your agent of choice can autonomously search, explore, and export.

▩

Plugin Architecture

Hot-swappable screeners, vector stores, exporters, and database connectors. Build your own without touching core logic.

☰

Multi-Format Export

CSV for spreadsheets, BibTeX for citation managers, RIS for reference tools, Markdown with PRISMA flow diagrams, and direct Obsidian export.

How It Works

From configuration to published results in four steps.

Configure

Define research anchors, technical terms, and scoring weights in config.json. The engine uses these as its compass.

Mine

The hunter sweeps all 7 academic APIs in parallel. Papers are fetched, deduplicated, and enriched with missing metadata.

Score

Every paper is evaluated by keyword and semantic screeners. Only those above your min_relevance_score threshold survive.

Report

Results are exported as rich reports (CSV, BibTeX, Markdown with PRISMA flow) and indexed into ChromaDB for semantic queries.

Quickstart

Clone, install, and start hunting in under two minutes.

$ git clone https://github.com/devfelipenunes/academic-hunter.git
$ cd academic-hunter
$ python install.py
$ source venv/bin/activate
$ academic-hunter
$ # Or, for MCP server mode:
$ academic-mcp
